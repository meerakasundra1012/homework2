COMP 484 HW - Structuring Strategy

1. Document outline

Each page follows this hierarchy:

  body
    header (role="banner")
      h1 Comp 484
      h2 id="page-heading"  [page title]
      section (aria-label="Main navigation")
        nav (role="navigation")  - list of page links
    main (role="main")
      section (role="region" aria-labelledby="page-heading")
        article  - first topic (header with h3 + section/content)
        article  - second topic
        ... more articles as needed
    aside (role="complementary")  - only on structure.html (W3C warning), sibling to main
    footer (role="contentinfo") - CSUN info

2. Structural tags

section
  Wraps the main content area once per page with role="region" for screen readers.
  Also wraps the nav in the header. Inside main, each major topic can have a section inside an article.

article
  One per self-contained topic (e.g. Introduction, History, HTML timeline). Multiple articles live inside the main content section.

nav
  Used only in the header for the top menu, with role="navigation".

aside
  Used only on structure.html for the W3C warning box. It is a direct child of body, with role="complementary" and aria-label="W3C note".

3. Semantic elements

header  Top of page: h1, h2, and the nav wrapped in a section. role="banner".
main    One per page; wraps the primary content section. role="main".
footer  Bottom of page with CSUN link. role="contentinfo".

4. ARIA roles

header              role="banner"
nav                 role="navigation"
section (around nav) aria-label="Main navigation"
main                role="main"
main content section role="region" aria-labelledby="page-heading"
aside               role="complementary" aria-label="W3C note"
footer              role="contentinfo"
